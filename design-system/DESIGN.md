# 主动健康管理平台 · 官方设计系统规范 (DESIGN.md)

本设计系统是面向健康管理机构的数字化平台而制定的，专为解决“监管大屏”的沉浸式数据监控与“管理后台”的高频业务操作双重场景设计。

---

## 1. 核心设计原则

* **高感知一致性 (OKLCH)**
  传统 RGB/HSL 颜色模型在亮度和对比度控制上缺乏统一性。本设计系统全站采用最新的 OKLCH 规范。OKLCH 在感知上更加均匀，保证了系统在深浅模式切换、复杂图表渲染时，各颜色的亮度和饱满度保持绝佳的视觉契合。
* **数据安全感与专业度**
  选用青蓝色（Primary）作为医疗科技主导色，传达信赖、理智与洁净的感知。
* **双通道分流**
  * **日常监管与高频处理**：采用亮色（SaaS Day Mode），确保长时间工作下的高信噪比与低眼睛疲劳度。
  * **宏观指标与大屏态势**：采用深色（Screen Night Mode），营造高级科技感与视觉对焦，让关键预警信号能够在暗色背景上“跳跃”出来。

## 2. 基础设计令牌 (Design Tokens)

### 2.1 色彩系统

#### A. 机构端后台 · 亮色 (Day Mode) 调色板
* **全局背景**：`oklch(0.99 0.005 200)`
  * *设计意图*：几乎全白且略带极淡的青蓝灰色，避开了纯白的刺眼感，提升界面整体的冷调质感。
* **前景色/主文本**：`oklch(0.2 0.03 220)`
  * *设计意图*：深蓝黑色，比纯黑更具灵动性。
* **品牌主色 (Primary)**：`oklch(0.55 0.13 195)`
  * *设计意图*：明快而不刺眼的医疗青蓝色，传达专业、可靠。
* **交互背景 (Secondary / Muted)**：`oklch(0.96 0.01 200)`
  * *设计意图*：浅灰蓝色，常用于输入框、按钮 Hover 态及卡片背景过渡。

#### B. 监管大屏 · 暗色 (Big Screen Mode) 调色板
* **大屏底色 (Screen BG)**：`oklch(0.18 0.04 230)`
  * *设计意图*：极其深沉的星空蓝灰色，为大屏提供暗色背景底盘，大幅降低发光显示屏在室内环境下的刺眼度。
* **大屏面板 (Screen Panel)**：`oklch(0.22 0.05 230)`
  * *设计意图*：略高于大屏底色，用于浮空卡片。
* **大屏描边 (Screen Border)**：`oklch(0.4 0.08 200 / 30%)`
  * *设计意图*：30% 透明度的灰蓝色，实现精细微光的物理切分。
* **大屏高亮 (Screen Accent)**：`oklch(0.78 0.18 195)`
  * *设计意图*：明亮冰蓝色，用于科技发光线条、指针及焦点高光。

#### C. 警情状态与图表色 (Feedback & Charts)
为了在浅色后台与深色大屏中同时保持高辨识度，状态色被精心映射在 OKLCH 的相同色相（Hue）内，保证情感共鸣：
* **成功/在线 (Success)**：`oklch(0.7 0.16 155)` —— 薄荷翠绿，寓意生命力与状态正常。
* **警告/低电 (Warning)**：`oklch(0.78 0.16 75)` —— 琥珀亮黄，警醒而不刺眼。
* **危险/红色警报 (Danger)**：`oklch(0.62 0.23 22)` —— 绯红警戒，最高等级的视觉对焦。
* **提示/信息 (Info)**：`oklch(0.62 0.14 230)` —— 天空深蓝，用于辅助系统提示或普通日志。

## 3. 材质与视觉特效 (Materials & Effects)

### 3.1 科技玻璃拟态 (`.screen-panel`)
在大屏设计中，面板不再是死板的单色，而是具备微弱折射的渐变玻璃体。
```css
background: linear-gradient(
  160deg, 
  color-mix(in oklab, var(--screen-panel) 90%, transparent), 
  color-mix(in oklab, var(--screen-bg) 90%, transparent)
);
border: 1px solid var(--screen-border);
box-shadow: 
  inset 0 1px 0 oklch(1 0 0 / 4%), /* 顶部边缘的物理微反光 */
  0 0 0 1px oklch(1 0 0 / 2%);
```

### 3.2 科技背景网格 (`.grid-bg`)
为了突破大屏黑色背景的单调感，采用 `oklch(1 0 0 / 4%)`（即 4% 不透明度的白色）绘制 32px 间隔的正交网格，带来微弱的精密工业感。

### 3.3 霓虹呼吸发光 (`.screen-glow`)
核心卡片（如高危待处理预警）会投下基于冰蓝色（Screen Accent）的半透明漫反射投影，营造有目的性的动态发光效果。

## 4. 排版与对齐规范 (Typography)

* **数字对齐 (Tabular Numbers)**
  由于系统涉及大量实时变动的体征数据（心率、血压、血氧）、时钟、编号、电话，因此全局声明：
  `font-feature-settings: "tnum" 1, "ss01" 1`
  该属性会强制浏览器将字符集中除正文外的所有阿拉伯数字采用“等宽（Tabular）”方式渲染。这可以有效避免在数据或时钟更新时，文本产生左右抖动与闪烁，极大改善了专业监控下的阅读体验。
* **大屏字间距 (Title Spacing)**
  主标题使用 `tracking-[0.3em]` 宽排字距，提升仪式感和科技底盘的稳定感。

## 5. 组件圆角层级 (Radius Levels)

系统圆角采用基于基准变量的层级递增规则，使得从细小微件到大型容器的弧度自然过渡：
* **小组件圆角 (Radius SM - 6px)**：常用于状态徽章（Badge）、多选框（Checkbox）及小型提示泡（Tooltip）。
* **常规组件圆角 (Radius MD - 8px)**：常用于表单输入框、按钮（Button）、下拉菜单（Dropdown）。
* **卡片/面板圆角 (Radius LG - 10px)**：常用于日常 SaaS 卡片容器（Card）。
* **巨型大屏容器 (Radius XL - 14px)**：常用于大屏的浮动玻璃组件框。

---

## 6. 数据图表设计准则 (Data Visualization)

在 Recharts 渲染时，应遵循以下配色规范：
1. **单一时间序列趋势**：使用 `var(--color-screen-accent)`（冰蓝）填充渐变区域，配合 2px 的主色折线，不使用折线节点圆点，利用无节点曲线维持图表洁净度。
2. **多指标对比（如心血管、睡眠、情绪）**：
   * 系列一（脑血管）：`var(--color-chart-4)` (警报绯红)
   * 系列二（睡眠健康）：`var(--color-chart-1)` (冰蓝)
   * 系列三（情绪压力）：`var(--color-chart-5)` (紫罗兰)
3. **占比雷达或饼图**：采用无描边多切片模式，切片间留 `paddingAngle={3}` 的微小物理缝隙，展现模块化的科技拼接感。
